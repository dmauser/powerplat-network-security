### Task 2: Network ACL Bypass Hardening
- **Decision:** Change `networkAcls.bypass` from `'AzureServices'` to `'None'` on Key Vault and Storage
- **Rationale:** The `bypass` setting is a public-endpoint firewall rule. With `publicNetworkAccess = 'Disabled'`, the public endpoint never accepts traffic, making any bypass value functionally inert. Setting `'None'` removes misleading exception semantics and explicitly signals defense-in-depth intent. No Microsoft Learn documentation describes a Power Platform VNet support scenario requiring `AzureServices` bypass with public access disabled.
- **Files Modified:**
  - `infra/modules/keyvault.bicep` тАФ Updated bypass + added inline rationale comment
  - `infra/modules/storage.bicep` тАФ Updated bypass + added inline rationale comment
